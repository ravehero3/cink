# UFO Sport Admin Dashboard - Complete Development Prompt

Build a comprehensive admin dashboard for the existing UFO Sport e-shop following the **exact same design principles** as the main site.

## CRITICAL: Design System Consistency

**USE THE SAME STRICT DESIGN RULES:**
- **Colors**: ONLY pure black (#000000) and pure white (#FFFFFF)
- **NO grays, shadows, gradients, or any other colors**
- **Typography**: Arial/Helvetica, same sizes as main site
  - Body: 14px
  - Headers: 18px  
  - Titles: 24px
- **Lines**: Always 1px thick, pure black
- **Spacing**: 16px, 24px, 32px increments
- **Buttons**: Black bg + white text OR white bg + black text with 1px border
- **All rectangles**: 1px black border

### Middleware Protection

```typescript
// middleware.ts
export function middleware(request: NextRequest) {
  const path = request.nextUrl.pathname;
  
  // Protect all /admin routes
  if (path.startsWith('/admin')) {
    const token = await getToken({ req: request });
    
    // Not logged in
    if (!token) {
      return NextResponse.redirect(new URL('/admin/login', request.url));
    }
    
    // Not admin role
    if (token.role !== 'ADMIN') {
      return NextResponse.redirect(new URL('/', request.url));
    }
  }
  
  return NextResponse.next();
}

export const config = {
  matcher: ['/admin/:path*']
};
```

## Security Measures

1. **Authentication:**
   - Session-based with NextAuth.js
   - Secure httpOnly cookies
   - CSRF protection

2. **Authorization:**
   - Middleware checks admin role
   - API endpoints verify admin role
   - Activity logging for audit trail

3. **Data Protection:**
   - Encrypt sensitive settings (API keys)
   - Use environment variables for secrets
   - Sanitize all user inputs
   - Parameterized queries (Prisma)

4. **Rate Limiting:**
   - Max 100 requests per minute per admin
   - Slower rate for login attempts (5 per 15 min)

5. **Input Validation:**
   - Server-side validation (never trust client)
   - Zod schemas for all inputs
   - File upload restrictions (type, size)

## Performance Optimization

1. **Pagination:**
   - All lists paginated (20-50 items)
   - Lazy load images
   - Virtual scrolling for large lists

2. **Caching:**
   - Cache dashboard stats (5 min TTL)
   - Cache product/category lists
   - Revalidate on mutations

3. **Database:**
   - Index frequently queried fields
   - Connection pooling
   - Optimize queries (select only needed fields)

4. **File Uploads:**
   - Client-side image compression
   - Upload directly to Cloudinary
   - Progress indicators
