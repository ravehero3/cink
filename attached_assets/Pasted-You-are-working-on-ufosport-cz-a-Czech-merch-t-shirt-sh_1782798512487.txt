You are working on ufosport.cz — a Czech merch/t-shirt shop where customers order custom t-shirts for delivery. The entire site is in Czech language.

TECH STACK — DO NOT CHANGE OR UPGRADE ANY OF THESE:
- Framework: Next.js 14 (Pages Router pattern, NOT App Router)
- Frontend: React 18, Tailwind CSS 3, Zustand (state), Lucide React (icons), Recharts (admin charts)
- Database: PostgreSQL with Prisma ORM v6
- Auth: NextAuth.js v4
- Payments: GoPay (Czech payment gateway)
- Image hosting: Cloudinary, UploadThing
- Email: Resend
- Excel export: xlsx
- PDF generation: jsPDF

SCRIPTS:
- Dev: npm run dev (next dev on port 5000, bound to 0.0.0.0)
- Build: npm run build (prisma generate && prisma db push && next build)
- Start: npm run start (next start on port 5000)
- DB commands: npm run db:push, db:generate, db:studio, db:seed, db:migrate
- Server listens on port 5000

FILE STRUCTURE:
- Pages: pages/ or app/ (Next.js routing)
- Components: components/
- API routes: pages/api/ or app/api/
- Prisma schema: prisma/schema.prisma
- Database: PostgreSQL via DATABASE_URL env var

CRITICAL RULES:
1. NEVER upgrade Next.js from 14 to 15+ — it has breaking changes
2. NEVER upgrade Prisma without checking migration compatibility
3. NEVER switch from Prisma to Drizzle or any other ORM
4. NEVER upgrade React from 18 to 19
5. NEVER change the port from 5000
6. NEVER modify prisma/schema.prisma without running db:generate after
7. All user-facing text must be in CZECH language
8. After changing the Prisma schema, always run: npx prisma generate
9. The admin dashboard uses Recharts — don't replace it with another charting library
10. GoPay integration handles CZK currency — don't change currency settings

COMMON PITFALLS TO AVOID:
- Don't use App Router patterns (layout.tsx, page.tsx) if the project uses Pages Router (or vice versa) — check which one it uses first
- Don't forget to run prisma generate after schema changes
- Don't break the GoPay callback/redirect URLs
- Don't remove the postinstall script (prisma generate)
- Prisma needs the DATABASE_URL environment variable — don't delete it from .env
