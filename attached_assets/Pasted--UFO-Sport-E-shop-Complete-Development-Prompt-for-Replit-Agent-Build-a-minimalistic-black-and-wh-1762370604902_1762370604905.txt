# UFO Sport E-shop - Complete Development Prompt for Replit Agent

Build a minimalistic black-and-white e-commerce website for UFO Sport (ufosport.cz) with the following exact specifications:

## Tech Stack
- **Framework**: Next.js 14 with App Router and TypeScript
- **Styling**: Tailwind CSS (customized for strict black/white palette)
- **Database**: PostgreSQL with Prisma ORM
- **Authentication**: NextAuth.js (for both users and admin)
- **Payment**: GoPay integration (Czech payment gateway)
- **Email**: Resend API for transactional emails
- **Shipping**: ZÃ¡silkovna (Packeta) API integration
- **File Storage**: Cloudinary for product images

---

## CRITICAL DESIGN SYSTEM

### Color Palette (STRICT - NO OTHER COLORS)
- **Background**: Pure white (#FFFFFF)
- **Text/Lines/Borders**: Pure black (#000000)
- **No grays, no shadows, no gradients**

### Typography
- **Font**: System font stack (Arial, Helvetica, sans-serif) for speed
- **Sizes**: Keep all text quite small for minimalistic look
  - Body text: 14px
  - Product names: 16px
  - Headers: 18px
  - Main titles: 24px
- **Weight**: Regular (400) for most text, Bold (700) for emphasis only
- **Line height**: 1.4 for readability

### Layout System
- **Lines**: All separating lines are exactly 1px thick, pure black
- **Spacing**: Consistent padding/margins (use 16px, 24px, 32px increments)
- **Grid**: 4 products per row on desktop (responsive: 2 on tablet, 1 on mobile)
- **All rectangles**: 1px black border
- **Buttons**: Only black or white with opposite text color

### Header Height Reference
- Header1 height = 60px (base unit)
- Header2 height = 60px
- Titlebar height = 240px (4x header)
- Videopromo height = 360px (6x header)
- Infobar height = 60px (same as header1)
- Footer2 height = 60px (same as header1)

## DATABASE SCHEMA

### Users Table
```prisma
model User {
  id            String    @id @default(uuid())
  email         String    @unique
  password      String    // Hashed with bcrypt
  name          String?
  phone         String?
  role          Role      @default(USER) // USER or ADMIN
  civility      String?   // Mr, Miss, Mrs, Ms, Mx
  newsletterSubscribed Boolean @default(false)
  savedProducts String[]  // Array of product IDs
  orders        Order[]
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt
}

enum Role {
  USER
  ADMIN
}
```

### Products Table
```prisma
model Product {
  id          String    @id @default(uuid())
  name        String
  slug        String    @unique
  description String    @db.Text
  price       Decimal   @db.Decimal(10, 2)
  category    String    // VOODOO808, SPACE LOVE, RECREATION WELLNESS, T SHIRT GALLERY
  color       String    // "black" or "white" (for filtering)
  images      String[]  // Array of Cloudinary URLs
  videoUrl    String?   // Optional promo video URL
  sizes       Json      // {S: 5, M: 10, L: 8, XL: 3, XXL: 2} - size and stock count
  isVisible   Boolean   @default(true)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}
```

## PAYMENT FLOW (GoPay Integration)

### Creating Payment:
1. User clicks ZAPLATIT
2. Backend creates order in database (status: PENDING)
3. Backend calls GoPay API to create payment:
   - Amount: total price in halÃ©Å™e (multiply by 100)
   - Currency: CZK
   - Order number
   - Return URL: `https://ufosport.cz/potvrzeni/[orderNumber]`
   - Notification URL: `https://ufosport.cz/api/gopay/webhook`
   - Items: list of products
4. Backend receives GoPay payment ID and gateway URL
5. Backend updates order with payment ID
6. Frontend redirects user to GoPay gateway URL

### After Payment:
- User completes payment on GoPay
- GoPay redirects to return URL
- Show order confirmation page
- GoPay sends webhook notification to backend
- Backend verifies payment status
- Backend updates order status to PAID
- Backend sends confirmation email

### GoPay Webhook Endpoint:
**URL:** `/api/gopay/webhook`
**Method:** POST
**Action:**
- Verify webhook signature
- Get payment status from GoPay
- Update order payment status
- If PAID â†’ trigger confirmation email
- Return 200 OK
